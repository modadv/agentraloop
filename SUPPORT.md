# Support

Use the right channel so maintainers can respond efficiently.

## Where to Ask

- Bug reports: open a GitHub issue with the bug report template.
- Feature requests: open a GitHub issue with the feature request template.
- Security concerns: follow [SECURITY.md](./SECURITY.md) and do not post exploit details publicly.

## Before Opening an Issue

- Read [README.md](./README.md) for setup and common commands.
- Read [CONTRIBUTING.md](./CONTRIBUTING.md) for development and verification expectations.
- Confirm the issue is reproducible on the current `main` branch when possible.
- Include exact commands, logs, screenshots, and environment details.

## Maintainer Expectations

Maintainers may close issues that:

- do not include enough detail to reproduce the problem
- request private support without project-level relevance
- duplicate existing issues or documented behavior
